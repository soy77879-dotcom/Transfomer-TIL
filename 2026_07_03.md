# Today I Learned (2026-07-06)

## 오늘 한 일

Transformer의 기본 개념과 전체 동작 흐름을 정리했다.

기존 Seq2Seq 모델이 가진 한계를 먼저 살펴보고, 그 한계를 Attention이 어떻게 보완하는지 공부했다. 이후 Transformer의 핵심 구조인 Self-Attention, Multi-Head Attention, 인코더, 디코더를 차례대로 정리했다. 단순히 용어를 외우기보다 Transformer가 왜 등장했는지, 그리고 문장 안의 단어 관계를 어떤 방식으로 계산하는지 이해하는 데 집중했다.

## 배운 내용

### 1. Transformer란?

Transformer는 2017년에 발표된 논문 *Attention Is All You Need*에서 제안된 딥러닝 모델 구조다. 자연어 처리에서 자주 쓰이던 RNN, LSTM, GRU와 달리 단어를 순서대로 하나씩 처리하지 않는다. 대신 문장 안의 단어들이 서로 얼마나 관련 있는지 Attention으로 계산한다.

Transformer의 핵심은 “문장을 순서대로 읽는 것”이 아니라 “문장 안의 단어 관계를 한 번에 계산하는 것”이다. 이 방식 덕분에 병렬 처리가 가능하고, 긴 문장에서도 멀리 떨어진 단어 사이의 관계를 더 잘 다룰 수 있다.

### 2. 기존 Seq2Seq 모델의 한계

Transformer를 이해하려면 먼저 기존 Seq2Seq 모델의 한계를 알아야 한다.

Seq2Seq 모델은 보통 인코더와 디코더로 구성된다.

- 인코더는 입력 문장을 하나의 벡터로 압축한다.
- 디코더는 그 벡터를 이용해 출력 문장을 만든다.

이 방식은 짧은 문장에서는 잘 동작할 수 있지만, 문장이 길어지면 한계가 드러난다. 입력 문장의 모든 정보를 하나의 벡터에 담기 어렵고, 앞쪽 단어의 정보가 뒤쪽으로 전달되는 과정에서 약해질 수 있다.

예를 들어 다음 문장을 생각해볼 수 있다.

```text
그 동물은 길을 건너지 않았다. 왜냐하면 그것은 너무 피곤했기 때문이다.
```

여기서 “그것”은 “길”이 아니라 “동물”을 가리킨다. 사람은 문맥을 보고 쉽게 이해하지만, 모델은 문장 안의 단어들이 어떤 관계를 갖는지 계산해야 한다. Attention은 이런 관계를 직접 계산하기 위해 사용된다.

### 3. Attention이 하는 일

Attention은 현재 단어를 이해할 때 문장 안의 어떤 단어를 더 중요하게 볼지 정하는 방법이다.

예를 들어 “그것”이라는 단어를 이해하려면 앞에 나온 “동물”과 “길” 중 어느 단어와 더 관련이 있는지 판단해야 한다. Attention은 단어 사이의 관련도를 점수로 계산한 뒤, 관련도가 높은 단어의 정보를 더 많이 반영한다.

Transformer에서는 Attention을 설명할 때 Query, Key, Value라는 세 가지 개념을 사용한다.

| 용어 | 쉬운 의미 |
| --- | --- |
| Query | 현재 단어가 찾고 싶은 정보 |
| Key | 비교 대상이 되는 정보 |
| Value | 실제로 가져와서 사용할 정보 |

쉽게 말하면 Query는 “무엇을 찾는가”, Key는 “무엇과 비교하는가”, Value는 “실제로 가져올 내용”이다.

### 4. Scaled Dot-Product Attention

Transformer에서 기본적으로 사용하는 Attention 계산 방식은 Scaled Dot-Product Attention이다.

```text
Attention(Q, K, V) = softmax((QK^T) / sqrt(d_k))V
```

계산 흐름은 다음과 같다.

1. Query와 Key를 곱해 단어 사이의 관련도 점수를 구한다.
2. 점수가 너무 커지지 않도록 `sqrt(d_k)`로 나눈다.
3. Softmax를 적용해 관련도 점수를 확률처럼 바꾼다.
4. 그 값을 Value에 곱해 최종 문맥 벡터를 만든다.

이 과정에서 문장 안의 모든 단어 쌍 사이의 관계를 계산한다. 그래서 Transformer는 RNN처럼 단어를 하나씩 순서대로 처리하지 않아도 전체 문맥을 한 번에 볼 수 있다.

### 5. Self-Attention

Self-Attention은 같은 문장 안에서 단어들이 서로를 참고하는 Attention이다. 한 문장의 단어들이 서로를 보면서 자신의 의미를 다시 정리하는 과정이라고 볼 수 있다.

입력 문장 벡터를 `X`라고 하면 Query, Key, Value는 다음처럼 만들어진다.

```text
Q = XW_Q
K = XW_K
V = XW_V
```

Q, K, V는 모두 같은 입력 `X`에서 만들어진다. 하지만 각각 다른 가중치 행렬을 거치므로 서로 다른 역할을 한다.

Self-Attention을 사용하면 각 단어는 자기 자신만 보는 것이 아니라 문장 안의 다른 단어들도 함께 참고한다. 이 덕분에 대명사가 어떤 명사를 가리키는지, 동사가 어떤 주어와 연결되는지 같은 문맥 정보를 표현에 반영할 수 있다.

### 6. Multi-Head Attention

Multi-Head Attention은 Attention을 여러 개로 나누어 동시에 계산하는 방법이다.

Attention을 한 번만 수행하면 문장을 한 가지 관점에서만 보게 된다. 하지만 여러 head를 사용하면 각 head가 서로 다른 관계를 볼 수 있다.

예를 들면 다음과 같다.

- 어떤 head는 주어와 동사의 관계를 볼 수 있다.
- 어떤 head는 대명사와 명사의 관계를 볼 수 있다.
- 어떤 head는 가까운 단어 사이의 관계를 볼 수 있다.

전체 흐름은 다음과 같다.

1. 입력에서 여러 개의 Query, Key, Value를 만든다.
2. 각 head가 Attention을 따로 계산한다.
3. 여러 head의 결과를 하나로 합친다.
4. 다시 선형 변환을 거쳐 최종 벡터를 만든다.

Multi-Head Attention은 같은 문장을 여러 관점에서 읽는 효과를 만든다.

### 7. 입력 처리 과정

Transformer에 문장을 넣기 전에는 임베딩과 위치 정보 추가 과정이 필요하다.

모델은 문자를 그대로 이해하지 못한다. 그래서 단어 또는 토큰을 숫자 벡터로 바꾼다. 이 과정을 임베딩이라고 부른다.

하지만 Transformer는 단어를 순서대로 읽지 않기 때문에 단어의 위치 정보를 따로 알려줘야 한다. 이를 위해 포지셔널 인코딩을 사용한다. 포지셔널 인코딩은 각 단어가 문장에서 어느 위치에 있는지 나타내는 벡터다.

대표적인 식은 다음과 같다.

```text
PE(pos, 2i)     = sin(pos / 10000^(2i / d_model))
PE(pos, 2i + 1) = cos(pos / 10000^(2i / d_model))
```

`pos`는 단어의 위치이고, `i`는 벡터 차원의 번호다. 짝수 차원에는 sine, 홀수 차원에는 cosine을 사용한다.

### 8. 인코더 구조

인코더는 입력 문장을 문맥이 반영된 벡터로 바꾸는 부분이다.

하나의 인코더 층은 두 가지 핵심 부분으로 구성된다.

1. 멀티헤드 셀프 어텐션
2. 위치별 피드포워드 신경망

각 부분 뒤에는 Add & Norm이 붙는다.

- Add는 잔차 연결을 의미한다.
- Norm은 층 정규화를 의미한다.

인코더의 흐름은 다음과 같다.

```text
입력
  -> 멀티헤드 셀프 어텐션
  -> Add & Norm
  -> 위치별 피드포워드 신경망
  -> Add & Norm
  -> 출력
```

멀티헤드 셀프 어텐션에서는 입력 문장의 모든 단어가 서로를 참고한다. 그 결과 각 단어 벡터는 단순한 단어 의미가 아니라 문맥이 반영된 의미를 갖게 된다.

위치별 피드포워드 신경망은 각 단어 벡터를 한 번 더 변환한다. Attention이 단어 사이의 관계를 반영한다면, 피드포워드 신경망은 각 단어 표현을 더 풍부하게 만드는 역할을 한다.

### 9. 디코더 구조

디코더는 인코더의 출력 정보를 참고해 새로운 문장을 생성하는 부분이다.

하나의 디코더 층은 세 가지 핵심 부분으로 구성된다.

1. 마스크드 셀프 어텐션
2. 인코더-디코더 어텐션
3. 위치별 피드포워드 신경망

디코더의 흐름은 다음과 같다.

```text
출력 문장 입력
  -> 마스크드 셀프 어텐션
  -> Add & Norm
  -> 인코더-디코더 어텐션
  -> Add & Norm
  -> 위치별 피드포워드 신경망
  -> Add & Norm
  -> 다음 단어 예측
```

디코더는 문장을 생성할 때 아직 나오지 않은 미래 단어를 보면 안 된다. 예를 들어 “나는 학교에 간다”라는 문장에서 “학교에”를 예측하는 시점이라면, 뒤에 나올 “간다”를 미리 참고하면 안 된다.

그래서 디코더의 첫 번째 어텐션에서는 미래 단어를 가리는 마스크를 사용한다. 이 방식을 마스크드 셀프 어텐션이라고 부른다.

```text
예측 위치: 3번째 단어
참고 가능: 1번째, 2번째, 3번째 단어
참고 불가: 4번째 이후 단어
```

인코더-디코더 어텐션은 디코더가 인코더의 출력 정보를 참고하는 부분이다.

```text
Query = 디코더의 현재 상태
Key   = 인코더의 출력
Value = 인코더의 출력
```

번역을 예로 들면, 디코더가 현재 단어를 만들 때 원문 문장의 어떤 부분을 참고해야 하는지 계산하는 과정이다.

### 10. Transformer의 전체 동작 과정

번역 작업을 기준으로 Transformer의 동작 과정을 정리하면 다음과 같다.

1. 입력 문장을 토큰으로 나눈다.
2. 각 토큰을 임베딩 벡터로 바꾼다.
3. 위치 정보를 더한다.
4. 인코더가 입력 문장 안의 단어 관계를 계산한다.
5. 인코더는 문맥이 반영된 벡터를 만든다.
6. 디코더는 시작 토큰을 입력받아 출력을 만들기 시작한다.
7. 디코더는 이전에 생성된 단어만 참고한다.
8. 디코더는 인코더의 출력도 함께 참고한다.
9. 다음 단어의 확률을 계산한다.
10. 가장 적절한 단어를 선택한다.
11. 종료 토큰이 나올 때까지 반복한다.

### 11. Transformer의 장점과 한계

Transformer의 장점은 다음과 같다.

- RNN보다 병렬 처리가 쉽다.
- 긴 문장에서도 멀리 떨어진 단어 사이의 관계를 계산할 수 있다.
- 문장 안의 단어 관계를 직접 반영할 수 있다.
- 여러 어텐션 head를 사용해 문장을 다양한 관점에서 볼 수 있다.
- 번역, 요약, 질의응답, 문장 생성 등 다양한 NLP 작업에 사용할 수 있다.

이 구조는 이후 BERT, GPT, T5 같은 모델의 기반이 되었다. BERT는 주로 인코더 구조를 사용하고, GPT는 주로 디코더 구조를 사용한다.

Transformer에도 한계가 있다.

- 모든 단어 쌍의 관계를 계산하므로 문장이 길어질수록 계산량이 커진다.
- 단어 순서를 직접 처리하지 않기 때문에 위치 정보를 따로 넣어야 한다.
- 큰 모델을 학습하려면 많은 데이터와 연산 자원이 필요하다.
- Attention 점수가 항상 사람이 이해하기 쉬운 의미를 보여주는 것은 아니다.

특히 Self-Attention은 문장 길이를 `n`이라고 할 때 계산량이 대략 `O(n^2)`으로 증가한다. 긴 문서를 처리할 때 메모리 사용량이 커지는 이유가 여기에 있다.

## 어려웠던 점

Query, Key, Value라는 용어가 처음에는 직관적으로 느껴지지 않았다. 이름만 보면 각각이 어떤 역할을 하는지 바로 연결되지 않았다.

또 Self-Attention, Multi-Head Attention, 마스크드 셀프 어텐션처럼 비슷한 이름의 개념이 많이 나와서 처음에는 구분이 헷갈렸다. 특히 인코더의 셀프 어텐션과 디코더의 마스크드 셀프 어텐션이 무엇이 다른지 따로 정리할 필요가 있었다.

포지셔널 인코딩도 어려웠다. Transformer가 단어를 순서대로 읽지 않는다는 점은 이해했지만, 위치 정보를 sine과 cosine으로 만들어 더한다는 방식은 처음에 바로 와닿지 않았다.

## 해결 방법

Query, Key, Value를 단순한 비유로 다시 정리했다.

- Query는 현재 단어가 찾고 싶은 정보다.
- Key는 비교 대상이 되는 정보다.
- Value는 실제로 가져와서 사용할 정보다.

이렇게 생각하니 Attention이 “현재 단어가 문장 안의 어떤 단어를 참고할지 정하는 과정”이라는 점이 더 잘 이해되었다.

Self-Attention과 마스크드 셀프 어텐션은 차이를 중심으로 정리했다.

- Self-Attention은 문장 안의 단어들이 서로를 모두 참고한다.
- 마스크드 셀프 어텐션은 디코더에서 미래 단어를 보지 못하게 막는다.

포지셔널 인코딩은 수식을 완전히 외우기보다, “Transformer에게 단어의 순서를 알려주는 장치”라고 먼저 이해했다. 이후 sine과 cosine을 사용하는 이유는 각 위치마다 서로 다른 패턴을 만들기 위한 방법으로 정리했다.

## 회고

오늘 Transformer를 정리하면서 가장 크게 느낀 점은, Transformer가 단순히 Attention을 추가한 모델이 아니라 Attention을 중심으로 전체 구조를 다시 설계한 모델이라는 점이다.

처음에는 Query, Key, Value, Self-Attention, Multi-Head Attention 같은 용어가 한꺼번에 나와서 복잡하게 느껴졌다. 하지만 하나씩 풀어보니 결국 핵심 질문은 단순했다. “현재 단어를 이해하기 위해 문장 안의 어떤 단어를 얼마나 참고해야 하는가?”라는 질문이다. Attention은 이 질문에 대한 답을 점수로 계산하는 방법이고, Self-Attention은 이 과정을 같은 문장 안에서 수행하는 방식이다.

특히 셀프 어텐션을 이해하면서 Transformer가 왜 RNN 없이도 문장을 처리할 수 있는지 조금 더 명확해졌다. RNN은 단어를 순서대로 읽으면서 정보를 전달하지만, Transformer는 문장 안의 모든 단어 관계를 한 번에 계산한다. 그래서 병렬 처리가 가능하고, 멀리 떨어진 단어 사이의 관계도 직접 확인할 수 있다.

인코더와 디코더의 역할도 이전보다 정리되었다. 인코더는 입력 문장을 이해하기 위한 문맥 벡터를 만들고, 디코더는 그 벡터를 참고해 출력 문장을 생성한다. 디코더에서는 미래 단어를 보면 안 되기 때문에 마스크드 셀프 어텐션을 사용한다. 이 차이를 알고 나니 인코더의 셀프 어텐션과 디코더의 셀프 어텐션이 왜 다르게 동작해야 하는지도 이해할 수 있었다.

아직 수식까지 완전히 자유롭게 설명할 수 있는 단계는 아니다. 특히 `QK^T`, `sqrt(d_k)`, Softmax가 실제 행렬 연산에서 어떤 크기로 계산되는지는 더 연습이 필요하다. 하지만 오늘 정리를 통해 각 구성 요소가 왜 필요한지에 대한 큰 흐름은 잡혔다.

앞으로 BERT나 GPT를 공부할 때도 인코더, 디코더, 셀프 어텐션, 마스크, 포지셔널 인코딩 개념이 계속 등장한다. 오늘 정리한 내용을 기준점으로 삼고, 다음에는 실제 코드에서 Query, Key, Value 행렬이 어떻게 만들어지는지 직접 확인해보고 싶다.
